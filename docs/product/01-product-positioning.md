# MKM 产品定位文档

## 1. 产品一句话定位

MKM 是一个面向程序员和技术学习者的个人 Markdown 知识库工具，优先解决 Android 端查看、管理和整理 Markdown 文档不方便的问题，并逐步扩展为支持 AI 助手、Web 同步、公开知识库和 CLI 调用的全栈知识管理产品。

---

## 2. 产品参考对象

### 主要参考：Obsidian

MKM 可以参考 Obsidian 的以下能力：

- 本地优先的 Markdown 知识库理念
- 文件夹 / Vault 式知识库组织方式
- Markdown 阅读与编辑体验
- 双链、标签、知识图谱等知识管理能力
- 插件化扩展思路
- 多端同步和个人知识库生态

### MKM 与 Obsidian 的区别

| 维度 | Obsidian | MKM |
|---|---|---|
| 核心定位 | 通用型 Markdown 知识库 | 面向程序员的移动端优先知识库 |
| 初始重点 | 桌面端强编辑体验 | Android 端 Markdown 查看与轻编辑 |
| 用户群体 | 知识工作者、学生、研究者、写作者 | 程序员、技术学习者、AI Agent 用户 |
| AI 方向 | 主要依赖插件生态 | 内置 AI 总结、写作、待办生成、知识库问答 |
| CLI 能力 | 非核心能力 | 未来作为 AI Agent 调用入口 |
| 社区方向 | 插件和主题生态 | 公开技术知识库、评论、技术内容分享 |

> 注意：MKM 是参考 Obsidian 的产品理念和交互方式进行学习与再设计，不是直接修改或复制 Obsidian 应用。

## 3. 目标用户

### 3.1 核心用户

**程序员 / 技术学习者**

典型特征：

- 经常阅读 Markdown 格式的技术文档
- 有自己的技术笔记、代码片段、学习总结
- 会使用 GitHub、README、Wiki、博客、AI 工具
- 希望在手机上也能方便查看和整理技术资料
- 可能希望把自己的知识库分享给朋友或公开展示

### 3.2 初期用户

- 开发者本人
- 技术朋友
- 小范围测试用户
- 对 Markdown、Obsidian、AI 知识库感兴趣的人

### 3.3 未来用户

- Android 端重度学习者
- 经常移动端阅读技术文档的程序员
- 使用 AI Agent 辅助编码和知识整理的人
- 想公开分享自己技术知识库的人

## 4. 核心痛点

### 4.1 手机端 Markdown 查看体验差

当前很多 Android 文件管理器或普通文本编辑器可以打开 `.md` 文件，但体验通常不好：

- 不能正确渲染标题、列表、代码块、表格
- 代码块没有高亮
- 图片、链接、任务列表展示不友好
- 长文档阅读体验差

### 4.2 程序员知识资料分散

技术资料通常散落在：

- 本地 Markdown 文件
- GitHub README
- 网盘 / 云盘
- 个人博客
- AI 对话记录
- 待办工具
- 浏览器收藏夹

MKM 希望逐步把这些技术知识统一整理到一个个人知识库中。

### 4.3 移动端不适合重编辑，但适合轻整理

手机端不适合长篇写作和复杂排版，但适合：

- 查看文档
- 快速补充笔记
- 让 AI 总结内容
- 从文档中提取待办
- 标记收藏、归档、打卡
- 分享给别人阅读

因此 MKM 的 Android 端不追求一开始就成为完整 Markdown 编辑器，而是先做好阅读、轻编辑和 AI 辅助整理。

## 5. 产品核心价值

### 第一核心价值

**帮助程序员在 Android 手机上方便地查看、管理和整理自己的 Markdown 技术知识库。**

### 第二核心价值

**通过 AI 助手提升 Markdown 文档的整理效率，例如总结文档、生成笔记、提取待办、回答知识库问题。**

### 第三核心价值

**未来支持公开知识库，让技术用户可以分享自己的知识库内容，并围绕内容进行评论和交流。**

---

## 6. 产品原则

### 6.1 本地优先，云端增强

初期应保证用户即使没有复杂后端能力，也能在 Android 本地查看和管理 Markdown 文件。

后端主要提供：

- 用户登录
- 云端同步
- 多端访问
- 公开分享
- AI 能力统一接入

### 6.2 阅读优先，编辑其次

V1 的重点不是做一个完整 Markdown 编辑器，而是：

1. 能稳定打开 Markdown
2. 能优雅渲染 Markdown
3. 能管理文档列表
4. 能做基础编辑和保存

高级编辑能力、所见即所得、双链编辑等可以放到后续版本。

### 6.3 程序员友好

MKM 应重点支持程序员常用内容：

- 代码块
- 代码高亮
- 表格
- README 风格文档
- GitHub Flavored Markdown
- 技术标签
- 文档目录
- API 文档
- 命令行片段

### 6.4 AI 是助手，不是替代品

AI 功能不应该一开始喧宾夺主，而应该作为知识库的增强能力：

- 总结当前文档
- 根据文档生成待办
- 帮用户改写 Markdown
- 根据知识库回答问题
- 帮 AI Agent 通过 CLI 读取和写入知识库

---

## 7. 产品边界

### V1 不做

- 不做完整 Obsidian 级别插件系统
- 不做复杂知识图谱
- 不做多人实时协作
- 不做复杂富文本编辑器
- 不做完整社区系统
- 不做商业化付费功能

### V1 必须做好

- Android 端 Markdown 渲染
- 用户注册登录
- 文档创建、查看、编辑、删除
- Android / Web 通过后端访问同一批文档
- 基础本地缓存

---

## 8. 成功标准

### 学习项目成功标准

- 完成 Android + 后端 + Web 的完整闭环
- 熟悉 Spring Boot、PostgreSQL、JWT、Vue 3、Android 网络请求和本地缓存
- 能够独立运行和调试整个项目

### 产品雏形成功标准

- 自己愿意持续使用
- 技术朋友可以完成注册、创建文档、查看 Markdown
- Android 端 Markdown 阅读体验明显优于普通文本打开
- 未来可以自然扩展 AI、公开知识库、评论和 CLI

---

## 9. 当前结论

MKM 的主定位确定为：

> 面向程序员的移动端优先个人 Markdown 知识库。

Obsidian 是主要参考对象，但 MKM 初期不追求完整复刻 Obsidian，而是从 Android 端 Markdown 阅读与个人知识库管理切入，再逐步扩展 AI 助手、Web 同步、公开知识库和 CLI 能力。
