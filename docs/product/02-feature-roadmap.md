# MKM 功能路线图

## 1. 路线图原则

MKM 的目标不是一次性复刻 Obsidian，而是从一个可运行、可学习、可持续扩展的增强 MVP 开始，逐步演进成面向程序员的移动端优先 Markdown 知识库产品。

当前路线采用：

> **增强 MVP 路线：基础知识库 + 待办 + AI 提问 + 更多知识库 + 我的。**

这意味着 V1 就不只是一个简单 Markdown 阅读器，而是要形成一个完整的产品雏形。

---

## 2. 底部主导航设计

Android 端和 Web 端都可以围绕同一套核心模块组织。

底部 Toolbar / Bottom Navigation 建议包含 5 个一级入口：

| 位置 | 模块 | 名称建议 | 核心作用 |
|---|---|---|---|
| 1 | Markdown 文档 | Markdown / 文档 | 查看和管理自己的 Markdown 文档 |
| 2 | 待办 | 待办 | 从文档或 AI 中沉淀任务、打卡、计划 |
| 3 | AI 提问 | AI | 中间主入口，围绕当前知识库提问和生成内容 |
| 4 | 更多知识库 | 知识库 | 查看公开知识库、示例库、别人分享的内容 |
| 5 | 我的 | 我的 | 账号、设置、同步、个人信息 |

推荐底部导航顺序：

```text
Markdown    待办    AI    知识库    我的
```

其中 **AI 提问放在中间**，作为未来产品的核心增强入口。

---

## 3. V1：增强 MVP

### 3.1 V1 目标

V1 的目标是完成一个可以实际使用的产品雏形：

- 用户可以登录
- 用户可以管理自己的 Markdown 文档
- 用户可以在 Android 和 Web 端查看 Markdown
- 用户可以创建待办
- 用户可以向 AI 提问
- 用户可以看到“更多知识库”的基础入口
- 用户可以进入“我的”管理账号和设置

V1 的重点是：

> 建立完整信息架构和产品闭环，而不是把每个模块都做到很深。

---

## 4. V1 一级模块规划

## 4.1 Markdown 模块

### 模块定位

Markdown 模块是 MKM 的基础模块，负责个人文档的查看、创建、编辑和管理。

### V1 必做功能

- 文档列表
- 文档详情
- Markdown 渲染
- 新建文档
- 编辑文档
- 删除文档
- 云端同步到数据库
- 基础本地缓存

### V1 建议支持的 Markdown 能力

- 标题
- 粗体 / 斜体
- 列表
- 任务列表
- 引用
- 代码块
- 表格
- 链接
- 图片基础展示

### V1 暂不做

- 双向链接
- 知识图谱
- 插件系统
- 所见即所得编辑器
- 多人协同编辑

### 页面结构

```text
Markdown 首页
├── 文档列表
├── 搜索入口
├── 新建按钮
└── 文档详情页
    ├── Markdown 预览
    ├── 编辑模式
    ├── AI 总结入口
    └── 生成待办入口
```

---

## 4.2 待办模块

### 模块定位

待办模块用于承接用户从文档、学习计划、AI 对话中产生的任务。

### V1 必做功能

- 待办列表
- 新建待办
- 完成 / 取消完成
- 删除待办
- 基础优先级
- 截止时间可选
- 关联 Markdown 文档可选

### V1 可选功能

- 今日待办
- 已完成分组
- 从 Markdown 文档中手动创建待办
- AI 根据文档生成待办草稿

### V1 暂不做

- 复杂日历视图
- 团队任务协作
- 周期性任务
- 复杂习惯打卡统计

### 页面结构

```text
待办首页
├── 今日待办
├── 全部待办
├── 已完成
└── 新建待办弹窗 / 页面
```

---

## 4.3 AI 提问模块

### 模块定位

AI 模块是 MKM 的增强核心，负责帮助用户理解、总结和生成 Markdown 内容。

AI 入口放在底部导航中间，强调其重要性。

### V1 必做功能

- 基础 AI 聊天页面
- 用户输入问题
- AI 返回回答
- 支持对当前文档提问
- 支持总结当前 Markdown 文档
- 支持生成 Markdown 草稿
- 支持从 AI 回答创建待办

### V1 的 AI 范围

V1 不要求实现完整知识库 RAG，可以先从两个简单场景开始：

1. **当前文档上下文**
   - 用户在文档详情页点击“总结”
   - 后端把当前文档内容发送给 AI
   - AI 返回总结

2. **普通 AI 问答**
   - 用户在 AI 页面输入问题
   - 后端调用 AI 接口
   - 返回回答

### V1 暂不做

- 全量知识库向量检索
- 多文档复杂 RAG
- 长期记忆
- Agent 自主执行复杂任务
- 多模型切换

### 页面结构

```text
AI 首页
├── 对话列表 / 当前对话
├── 输入框
├── 快捷操作
│   ├── 总结文档
│   ├── 生成 Markdown
│   ├── 提取待办
│   └── 优化表达
└── 回答结果操作
    ├── 复制
    ├── 保存为 Markdown
    └── 创建待办
```

---

## 4.4 更多知识库模块

### 模块定位

更多知识库是未来公开知识库、示例库和社区内容的入口。

V1 不要求完整社区化，但需要预留信息架构。

### V1 必做功能

- 更多知识库首页
- 展示系统内置示例知识库
- 展示公开知识库列表的占位结构
- 查看公开 Markdown 文档详情

### V1 可选功能

- 用户将自己的文档设为公开
- 公开知识库详情页
- 收藏公开文档

### V1 暂不做

- 评论系统
- 点赞系统
- 关注用户
- 推荐算法
- 完整社区运营后台

### 页面结构

```text
更多知识库首页
├── 推荐知识库
├── 示例知识库
├── 公开文档列表
└── 知识库详情页
```

---

## 4.5 我的模块

### 模块定位

我的模块用于管理账号、同步、设置和产品信息。

### V1 必做功能

- 显示当前登录用户
- 退出登录
- 同步状态
- 基础设置
- 关于 MKM

### V1 可选功能

- 修改昵称
- 主题设置
- Markdown 渲染偏好
- AI API Key 设置入口

### V1 暂不做

- 会员订阅
- 支付管理
- 复杂个人主页
- 完整通知中心

### 页面结构

```text
我的
├── 用户信息
├── 同步状态
├── 设置
│   ├── 主题
│   ├── Markdown 渲染
│   └── AI 设置
├── 关于
└── 退出登录
```

## 5. 版本路线图

## V1：增强 MVP

### 核心目标

完成产品主结构和基础闭环。

### 包含模块

- Markdown 文档管理
- 待办管理
- AI 基础提问
- 更多知识库入口
- 我的设置
- 用户登录注册
- Android + Web + 后端 + 数据库闭环

### 成功标准

- 用户能注册登录
- 用户能创建和查看 Markdown 文档
- 用户能在 Android 端看到正常渲染效果
- 用户能在 Web 端访问同一份文档
- 用户能创建待办
- 用户能向 AI 提问并获得回答
- 用户能看到更多知识库入口和公开内容雏形

---

## V2：AI 增强版

### 核心目标

让 AI 真正与知识库结合，提高整理效率。

### 主要功能

- 当前文档总结
- 当前文档问答
- 根据文档生成待办
- AI 生成 Markdown 草稿
- AI 优化 Markdown 表达
- 对话记录保存
- AI 回答保存为文档

### 技术重点

- 后端 AI 服务封装
- Prompt 模板管理
- AI 调用日志
- 基础额度控制
- 文档上下文传递

---

## V3：知识库公开与互动

### 核心目标

从个人工具扩展为可分享的技术知识库平台。

### 主要功能

- 用户公开自己的知识库
- 公开文档浏览
- 评论功能
- 收藏功能
- 用户个人公开主页
- 知识库分类

### 技术重点

- 公开权限模型
- 评论数据模型
- 内容审核预留
- 公开页面 SEO 预留

---

## V4：CLI 与 AI Agent 调用

### 核心目标

让 MKM 可以被其他 AI Agent 或命令行工具调用。

### 主要功能

- CLI 登录
- CLI 查询文档
- CLI 创建文档
- CLI 更新文档
- CLI 创建待办
- CLI 调用 AI 总结文档
- 输出 JSON，方便 Agent 集成

### 示例命令

```bash
mkm login
mkm docs list
mkm docs get <id>
mkm docs create --title "学习笔记" --file note.md
mkm ai summarize --doc <id>
mkm todo create "阅读 Spring Security 文档"
```

## 7. 当前结论

MKM 的功能路线采用增强 MVP：

```text
V1：Markdown + 待办 + AI 基础提问 + 更多知识库入口 + 我的
V2：AI 深度增强
V3：公开知识库与评论互动
V4：CLI 与 AI Agent 集成
```

底部导航确定为：

```text
Markdown    待办    AI    知识库    我的
```

其中 AI 位于中间，是后续增强体验和产品差异化的重要入口。
