# MKM 功能路线图

## 1. 路线图原则

MKM 的目标不是一次性复刻 Obsidian，而是从一个可运行、可学习、可持续扩展的增强 MVP 开始，逐步演进成面向程序员的移动端优先 Markdown 知识库产品。

当前路线采用：

> **本地优先增强 MVP：知识库管理 + AI 助手 + 待办 + 多端同步。**

V1 不做社区化功能，专注本地知识库体验与 AI 增强。

---

## 2. Android 主导航设计

Android 端使用 **悬浮底部 3 Tab** 主导航：

| 位置 | 模块 | 核心作用 |
|---|---|---|
| 1 | Markdown | 知识库文件夹树 + 文档管理 + Markdown 阅读编辑 |
| 2 | AI | 多会话 AI 助手，支持文档总结/生成/问答 |
| 3 | 我的 | 账号、设置、同步、主题、AI Key 配置 |

底部 Tab 悬浮展示，不贴底实心栏。AI 入口位于中间，作为未来差异化核心。

### Markdown Toolbar（顶部工具栏）

```text
用户头像    搜索    日历（待办入口）    更多
```

- 用户头像：进入"我的"或显示账号信息
- 搜索：知识库内全文搜索（文件名 + 标签 + 正文）
- 日历：点击展开弹窗，显示日历视图 + 跳转待办页入口
- 更多：文件操作菜单

### 右滑抽屉

```text
用户基本信息
──────────────
知识库信息
新建知识库
打开知识库（App 内 / 系统文件夹）
切换知识库
知识库设置
```

## 3. 知识库设计

### 3.1 知识库类型

| 类型 | 说明 |
|---|---|
| App 内知识库 | 默认类型，存储在 App 私有目录，稳定安全 |
| 系统文件夹知识库 | 用户选择手机目录，直接读取 `.md` 文件，类似 Obsidian Vault |

### 3.2 多知识库隔离规则

```text
独立：Markdown 文件结构
共享：待办、AI 会话、我的设置
```

### 3.3 文件夹树结构

知识库首页展示目录树，支持展开/折叠文件夹：

```text
知识库根目录
├── Android/
│   ├── Room.md
│   └── Retrofit.md
├── Backend/
│   └── Spring Security.md
└── README.md
```

### 3.4 首页视图

支持卡片/列表双视图切换，默认卡片式。

## 4. V1：本地优先增强 MVP

### 4.1 V1 目标

```text
用户可以在 Android 端本地管理 Markdown 知识库
用户可以选择性将文档同步到云端
用户可以在 Android 和 Web 端查看/编辑同步文档
用户可以使用 AI 进行文档总结、问答、生成
用户可以从日历入口管理待办
```

---

## 4.2 Markdown 模块

### 必做功能

- 知识库文件夹树展示（展开/折叠）
- 卡片/列表视图切换
- 文档详情 + Markdown 渲染
- 带富文本快捷工具栏的编辑器（加粗/标题/列表/代码块等）
- 新建文件（在当前选中文件夹下创建）
- 新建文件夹
- 重命名、删除、移动、复制
- 导入、导出、分享 `.md` 文件
- 图片支持：本地相对路径 + 网络图片
- 附件支持：PDF / ZIP / 其他，单文件上限 10MB
- 完整离线可用
- 每篇文档独立控制云端同步开关
- 本地回收站（App 内知识库删除先进回收站）

### 标签功能

- 给文档打标签
- 创建/管理标签
- 按标签筛选文档列表

### 搜索功能

- 搜索范围：文件名 + 标签 + 正文内容
- 支持知识库内全文搜索

### 不做

- 双向链接
- 知识图谱
- 插件系统
- 所见即所得编辑器
- 多人协同编辑

---

## 4.3 AI 模块

### 可用条件

```text
必须登录 + 在"我的"中配置了 AI API Key
```

### Provider 配置

```text
单个 OpenAI 兼容配置
用户填写：Base URL / API Key / Model 名称
兼容：OpenAI / DeepSeek / Kimi / 通义等 OpenAI-compatible 服务
```

### 必做功能

- 多会话管理（新建/切换/删除，类似 ChatGPT 左侧列表）
- 对话历史云端同步
- 普通 AI 问答
- 当前文档总结
- 生成 Markdown 草稿
- 从 AI 回答提取待办
- AI 回答复制

### 不做

- 全量知识库向量检索
- 多 Provider 模板
- 多模型切换
- Agent 自主执行复杂任务
- 长期记忆

---

## 4.4 待办模块

### 入口

```text
Markdown Toolbar → 日历弹窗 → 待办页
```

### 字段

```text
标题
备注
日期（可选）
优先级（低/中/高）
完成状态
关联 Markdown 文档（可选）
```

### 来源

- 用户手动创建
- AI 提取生成

### 不做

- 独立 Tab 入口
- 复杂日历视图
- 周期性任务
- 团队任务协作
- 通知提醒

---

## 4.5 我的模块

### 完整功能范围

```text
用户信息展示
修改昵称 / 头像
退出登录
账号注销

AI Key 配置（Base URL / API Key / Model）

同步设置：
  同步状态总览（已同步/失败/待同步、上次同步时间）
  全局同步开关
  仅 Wi-Fi 同步
  冲突文档列表（逐条选择保留本地或云端）
  失败重试
  同步日志

主题设置：
  浅色 / 深色 / 跟随系统
  自定义主色调选择器

Markdown 渲染偏好：
  字体大小
  字体选择
  代码块主题（亮色/暗色）
  行间距
  代码语法高亮开关
  图片加载策略（Wi-Fi only / 总是加载）

清理缓存
通知设置
关于 MKM
```

## 5. V1 Web 范围

Web 端 V1 定位为辅助管理端。

### V1 必做

- 登录 / 注册
- 知识库文件夹树（与 Android 保持一致）
- 文档列表（卡片/列表切换）
- 文档详情 Markdown 渲染
- 新建 / 编辑文档
- 删除 / 重命名 / 移动文档
- 搜索（文件名 + 标签 + 正文）
- 标签管理
- 待办列表 + 新建 / 完成 / 删除待办

### V1 不做

- AI 功能
- 我的复杂设置
- 社区功能
- 通知

## 6. V1 后端范围

### V1 必做 API 能力

```text
Auth：注册、登录、JWT 鉴权、获取当前用户

Document：
  创建/获取/更新/删除文档
  获取我的文档列表（支持搜索/标签筛选）
  文件夹结构同步

Attachment：
  上传附件（≤10MB）
  获取附件下载地址（相对路径解析）

Todo：
  创建/获取/更新/删除待办
  标记完成/取消完成

AI：
  普通提问
  文档总结
  生成 Markdown 草稿
  提取待办

AI Conversation：
  对话记录云端同步
  多会话管理（新建/切换/删除）
```

### V1 不做

- 评论
- 点赞
- 社区公开功能
- 向量检索

### V3：协作与分享

- 文档分享链接（只读访问）

- CLI 登录
- CLI 文档 CRUD
- CLI 待办管理

```bash

mkm docs list
mkm docs get <id>
mkm docs create --title "学习笔记" --file note.md
mkm ai summarize --doc <id>
mkm todo create "阅读 Spring Security 文档"
```

## 9. 当前结论

MKM V1 采用 **本地优先增强 MVP**：

```text
Android 是主产品端（悬浮 3 Tab：Markdown / AI / 我的）
Backend 是数据和 AI 能力中心
Web 是辅助管理端（文档管理 + 待办）
社区功能 V1 不做
```

V1 核心体验链路：

```text
打开 App → 无需登录 → 本地新建知识库 → 创建 Markdown → 编辑保存 → 渲染查看
登录后 → 选择文档同步到云端 → Web 端查看 → Android/Web 数据一致
AI → 登录 + 配置 Key → 多会话问答 → 文档总结 → 提取待办
```
