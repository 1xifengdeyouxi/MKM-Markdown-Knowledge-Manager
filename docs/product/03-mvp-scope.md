# MKM V1 MVP 范围定义

## 1. MVP 策略

MKM V1 采用 **本地优先增强 MVP** 策略。

核心原因：

1. 项目的原始痛点来自 Android 端无法舒服查看 Markdown 文件。
2. 本地优先让用户无需注册即可立即体验核心价值。
3. 每篇文档独立控制是否同步云端，尊重用户数据主权。
4. Web 端和后端作为支撑能力，不抢 V1 的主线。

平台优先级：

```text
Android 体验优先 > 后端 API 稳定 > Web 基础可用
```

---

## 2. V1 总目标

> 用户可以在 Android 端本地管理 Markdown 知识库，选择性云端同步，使用 AI 问答和总结，通过日历入口管理待办，并通过 Web 端进行基础文档和待办管理。

V1 不追求功能全面，而追求核心闭环完整。V1 不做社区化功能。

---

## 3. V1 平台职责划分

| 平台 | V1 定位 | 职责 |
|---|---|---|
| Android | 主产品端 | 知识库管理、Markdown 编辑阅读、AI 助手、待办、设置 |
| Backend | 数据与能力中心 | 用户认证、文档云端存储、待办数据、AI 转发、附件存储 |
| Web | 辅助管理端 | 文档管理（含文件夹树）、待办 |
| CLI | 暂缓 | V4 再做，预留 API 设计即可 |

## 5. V1 模块范围

### 5.1 知识库模块

#### 知识库类型

| 类型 | 说明 |
|---|---|
| App 内知识库 | 默认类型，存储在 App 私有目录 |
| 系统文件夹知识库 | 用户选择手机目录，直接读取 `.md` 文件 |

#### 多知识库隔离规则

```text
独立：Markdown 文件结构
共享：待办、AI 会话、我的设置（全局）
```

#### 文件夹树展示

```text
知识库根目录
├── Android/
│   ├── Room.md
│   └── Retrofit.md
├── Backend/
│   └── Spring Security.md
└── README.md
```

支持展开/折叠文件夹，同时支持卡片/列表视图切换（默认卡片）。

---

### 5.2 Markdown 模块

#### 模块目标

让用户可以在 Android 端本地管理、查看、编辑 Markdown 知识库，支持附件，完整离线可用。

#### V1 必做

**文档管理**
- 知识库文件夹树展示
- 文档详情 Markdown 渲染
- 带富文本快捷工具栏的编辑器（加粗/斜体/标题/列表/代码块/链接等）
- 新建文件（在当前选中文件夹下创建，无选中则放根目录）
- 新建文件夹
- 重命名文件 / 文件夹
- 删除文件 / 文件夹
- 移动文件 / 文件夹
- 复制文件 / 文件夹
- 导入 `.md` 文件
- 导出 `.md` 文件
- 分享 `.md` 文件

**删除策略**
- App 内知识库：进入 App 内回收站，可恢复/彻底删除
- 系统文件夹知识库：删除前强提示风险

**标签**
- 给文档打标签
- 创建 / 管理标签
- 按标签筛选文档列表

**搜索**
- 范围：文件名 + 标签 + 正文内容全文搜索

**附件**
- 支持本地相对路径图片 + 网络图片预览
- 支持 PDF / ZIP / 其他文件作为附件展示和打开
- 单文件大小上限：10MB
- 超出 10MB：提示用户暂不支持同步，可保留本地引用
- 附件引用：本地保持相对路径；云端由后端/API 解析为下载地址，不改原始 Markdown 内容

**同步**
- 每篇文档独立控制"仅本地" / "云端同步"
- 有网时标记为"云端同步"的文档自动后台上传
- 检测到本地与云端都有修改时，弹窗让用户选择保留本地或云端版本

#### V1 不做

- 双向链接
- 知识图谱
- 插件系统
- 所见即所得编辑器
- 多人协同编辑
- Git 同步

#### 最小闭环

```text
打开 App（无需登录）→ 新建知识库 → 新建 Markdown → 编辑（带工具栏）→ 保存 → 查看渲染
登录后 → 文档开启云端同步 → Web 端看到同一篇文档
```

### 5.3 待办模块

#### 入口

```text
Markdown Toolbar → 日历图标 → 日历弹窗 → 进入待办页
```

#### 字段

```text
标题（必填）
备注（可选）
日期（可选）
优先级：低 / 中 / 高（可选）
完成状态：未完成 / 已完成
关联 Markdown 文档（可选）
```

#### 来源

- 用户手动创建
- AI 提取生成

#### V1 不做

- 独立 Tab 入口
- 复杂日历视图
- 周期性任务
- 团队任务协作
- 通知提醒

#### 最小闭环

```text
Toolbar → 日历弹窗 → 进入待办 → 创建待办 → 标记完成 → 查看完成状态
```

---

### 5.4 AI 模块

#### 可用条件

```text
必须登录 + 在"我的"中配置了 AI API Key
未配置时显示引导页
```

#### Provider 配置

```text
单个 OpenAI 兼容配置
用户填写：Base URL / API Key / Model 名称
兼容 OpenAI / DeepSeek / Kimi / 通义等 OpenAI-compatible 服务
Key 后端加密存储，返回前端脱敏显示
```

#### V1 必做

- 多会话管理（新建/切换/删除会话，类似 ChatGPT 左侧列表）
- 对话历史云端同步
- 普通 AI 问答
- 当前文档总结
- 生成 Markdown 草稿
- 从 AI 回答提取待办
- AI 回答复制

#### V1 不做

- 全量知识库向量检索
- 多 Provider 模板 / 多模型切换
- Agent 自主执行复杂任务
- 长期记忆

#### 最小闭环

```text
配置 Key → 进入 AI → 新建会话 → 输入问题 → 获得回答 → 复制/创建待办
```

```text
文档详情 → 点击 AI 总结 → AI 返回摘要 → 复制或保存
```

---

### 5.5 我的模块

#### 模块目标

提供账号、AI Key 配置、同步管理、主题设置、渲染偏好等完整设置中心。

#### V1 必做

**账号**
- 当前登录用户展示
- 修改昵称 / 头像
- 退出登录
- 账号注销

**AI 设置**
- AI Key 配置（Base URL / API Key / Model 名称）

**同步设置（完整版）**
- 同步状态总览（已同步/失败/待同步数量、上次同步时间）
- 全局同步开关
- 仅 Wi-Fi 同步开关
- 冲突文档列表（逐条选择保留本地或云端）
- 失败重试
- 同步日志

**主题设置**
- 浅色 / 深色 / 跟随系统
- 自定义主色调选择器

**Markdown 渲染偏好（完整版）**
- 字体大小
- 字体选择
- 代码块主题（亮色/暗色）
- 行间距
- 代码语法高亮开关
- 图片加载策略（Wi-Fi only / 总是加载）

**其他**
- 清理缓存
- 通知设置
- 关于 MKM

#### 最小闭环

```text
进入我的 → 配置 AI Key → 退出登录 → 跳转登录页
```

## 6. V1 Web 范围

Web 端 V1 定位为辅助管理端。

### V1 必做

- 登录 / 注册
- 知识库文件夹树（与 Android 结构一致）
- 文档列表（卡片/列表切换）
- 文档详情 Markdown 渲染
- 新建 / 编辑文档
- 删除 / 重命名 / 移动文档
- 搜索（文件名 + 标签 + 正文）
- 标签管理
- 待办列表
- 新建 / 完成 / 删除待办

### V1 不做

- AI 功能
- 我的复杂设置
- 社区功能
- 通知中心
- 复杂账号管理

### Web 最小闭环

```text
登录 → 查看知识库文件夹树 → 新建文档 → Android 同步后看到
```

## 7. V1 后端范围

### V1 必做 API 能力

#### Auth

- 注册
- 登录
- JWT 鉴权
- 获取当前用户信息

#### Document

- 创建文档
- 获取我的文档列表（支持搜索/标签筛选）
- 获取文档详情
- 更新文档
- 删除文档
- 文件夹结构同步

#### Attachment

- 上传附件（单文件 ≤ 10MB，PostgreSQL 二进制存储）
- 获取附件下载地址（根据文档路径+相对路径解析）

#### Todo

- 创建待办
- 获取待办列表
- 更新待办（状态/字段）
- 删除待办
- 标记完成 / 取消完成

#### AI

- Provider 配置（添加/获取/更新/删除，Key 加密存储）
- 普通提问
- 文档总结
- 生成 Markdown 草稿
- 提取待办

#### AI Conversation

- 新建会话
- 获取会话列表
- 获取会话消息记录
- 删除会话
- 添加消息记录

### V1 不做

- 评论
- 点赞
- 社区公开文档
- 向量检索

## 10. V1 验收标准

### 流程 1：Android 本地知识库闭环

```text
无需登录 → 新建知识库 → 新建 Markdown → 编辑（工具栏）→ 保存 → 查看渲染
```

### 流程 2：文件夹管理闭环

```text
新建文件夹 → 在文件夹内创建文档 → 重命名 → 移动到另一文件夹 → 删除（进回收站）
```

### 流程 3：云端同步闭环

```text
登录 → 开启文档云端同步 → Web 端看到同一篇文档 → 修改后 Android 同步更新
```

### 流程 4：待办闭环

```text
Toolbar 日历 → 进入待办 → 创建任务 → 设置优先级和日期 → 标记完成
```

### 流程 5：AI 问答闭环

```text
配置 AI Key → 进入 AI → 新建会话 → 输入问题 → 获得回答 → 切换历史会话
```

### 流程 6：文档总结闭环

```text
打开 Markdown → 点击 AI 总结 → 看到摘要 → 复制或提取为待办
```

### 流程 7：Web 文档管理闭环

```text
Web 登录 → 查看文件夹树 → 新建文档 → Android 刷新后看到
```

### 流程 8：同步冲突处理闭环

```text
本地和云端各有修改 → 弹窗提示冲突 → 选择保留本地或云端 → 确认后同步
```

---

## 11. 当前结论

MKM V1 采用 **本地优先增强 MVP**：

```text
Android 是主产品端（悬浮 3 Tab：Markdown / AI / 我的）
Backend 是数据和 AI 能力中心
Web 是辅助管理端（文档管理 + 待办）
社区功能 V1 不做，V3 再评估
```

V1 的核心体验链路：

```text
本地知识库 → Markdown 编辑渲染 → 选择性云端同步 → AI 助手 → 待办管理
```
