# MKM V1 MVP 范围定义

## 1. MVP 策略

MKM V1 采用 **Android 优先的增强 MVP** 策略。

核心原因：

1. 项目的原始痛点来自 Android 端无法舒服查看 Markdown 文件。
2. 产品差异化优先体现在移动端阅读和轻整理体验。
3. 用户本人更熟悉 Android，适合作为主战场推进。
4. Web 端和后端作为支撑能力，不抢 V1 的主线。

因此 V1 的平台优先级为：

```text
Android 体验优先 > 后端 API 稳定 > Web 基础可用
```

---

## 2. V1 总目标

V1 需要完成一个可真实使用的产品雏形：

> 用户可以在 Android 端登录，查看数据库中的 Markdown 文档，创建/编辑文档，使用基础 AI 提问和总结文档，管理简单待办，并通过 Web 端进行基础文档管理。

V1 不追求功能全面，而追求核心闭环完整。

---

## 3. V1 平台职责划分

| 平台 | V1 定位 | 职责 |
|---|---|---|
| Android | 主产品端 | Markdown 阅读、轻编辑、待办、AI 提问、知识库浏览、我的 |
| Backend | 数据与能力中心 | 用户认证、文档数据、待办数据、AI 调用、公开知识库数据 |
| Web | 辅助管理端 | 登录、查看文档、编辑文档、基础管理 |
| CLI | 暂缓 | V4 再做，预留 API 设计即可 |

---

## 4. V1 Android 范围

## 4.1 Android 底部导航

V1 Android 底部导航固定为 5 个入口：

```text
Markdown    待办    AI    知识库    我的
```

### 导航说明

| Tab | V1 要求 |
|---|---|
| Markdown | 必须可用，是 V1 最核心模块 |
| 待办 | 必须可用，但功能保持简单 |
| AI | 必须可用，先支持基础问答和文档总结 |
| 知识库 | 必须有入口，可先做公开知识库列表/示例内容 |
| 我的 | 必须可用，支持账号与设置 |

---

## 5. V1 模块范围

# 5.1 Markdown 模块

## 模块目标

让用户可以在 Android 端舒服地查看和管理自己的 Markdown 文档。

## V1 必做

- 文档列表
- 文档详情
- Markdown 渲染
- 新建文档
- 编辑文档
- 删除文档
- 文档保存到后端数据库
- Android 本地缓存最近文档
- 从列表进入详情
- 从详情进入编辑
- 从详情触发 AI 总结
- 从详情生成待办入口

## V1 可选

- 文档搜索
- 文档标签
- 文档分类 / 文件夹
- 收藏文档
- 最近浏览
- 文档分享链接

## V1 不做

- 双向链接
- 知识图谱
- 插件系统
- 多人协作
- 高级 Markdown 编辑器
- Git 同步

## 最小闭环

```text
登录 → 文档列表 → 新建 Markdown → 保存 → 查看渲染 → 编辑 → 再次保存
```

---

# 5.2 待办模块

## 模块目标

帮助用户把学习计划、文档中的行动项、AI 总结中的任务沉淀为待办。

## V1 必做

- 待办列表
- 新建待办
- 完成待办
- 取消完成
- 删除待办
- 待办保存到后端数据库
- 基础状态：未完成 / 已完成

## V1 可选

- 截止时间
- 优先级
- 关联文档
- 今日待办
- AI 生成待办草稿

## V1 不做

- 日历视图
- 周期性任务
- 复杂习惯打卡统计
- 团队协作任务
- 通知提醒

## 最小闭环

```text
进入待办 → 创建待办 → 标记完成 → 查看已完成状态
```

---

# 5.3 AI 模块

## 模块目标

提供一个简单但可用的 AI 助手，帮助用户围绕 Markdown 文档进行提问、总结和生成内容。

## V1 必做

- AI 提问页面
- 用户输入问题
- 后端调用 AI 服务
- 返回 AI 回答
- 当前文档总结
- 根据用户输入生成 Markdown 草稿
- AI 回答复制

## V1 可选

- AI 回答保存为 Markdown 文档
- 从 AI 回答生成待办
- Prompt 快捷按钮
- 对话历史保存

## V1 不做

- 全知识库 RAG
- 向量数据库
- 多模型切换
- Agent 自主操作
- 长期记忆
- 多轮复杂工具调用

## 最小闭环

```text
进入 AI → 输入问题 → 后端请求 AI → 返回回答 → 复制/保存结果
```

当前文档总结闭环：

```text
文档详情 → 点击总结 → AI 返回摘要 → 用户复制或保存
```

---

# 5.4 知识库模块

## 模块目标

为未来公开知识库和社区功能预留入口，同时 V1 提供简单可浏览内容。

## V1 必做

- 知识库首页
- 示例知识库列表
- 公开文档列表
- 查看公开文档详情

## V1 可选

- 用户将自己的文档设为公开
- 收藏公开文档
- 知识库分类
- 热门知识库占位

## V1 不做

- 评论
- 点赞
- 关注用户
- 推荐算法
- 内容审核后台
- 复杂个人主页

## 最小闭环

```text
进入知识库 → 查看公开文档列表 → 打开公开 Markdown → 正常渲染
```

---

# 5.5 我的模块

## 模块目标

提供账号、同步、设置和产品信息入口。

## V1 必做

- 当前登录用户展示
- 退出登录
- 同步状态展示
- 关于 MKM
- 基础设置入口

## V1 可选

- 主题切换
- Markdown 渲染设置
- AI Key 设置入口
- 清理缓存

## V1 不做

- 会员订阅
- 支付管理
- 通知中心
- 复杂账号资料编辑

## 最小闭环

```text
进入我的 → 查看用户信息 → 退出登录 → 回到登录页
```

---

## 6. V1 Web 范围

Web 端 V1 定位为辅助管理端，不追求完整移动端体验。

## V1 必做

- 登录 / 注册
- 文档列表
- 文档详情 Markdown 渲染
- 新建 / 编辑文档
- 删除文档

## V1 可选

- 待办列表
- AI 提问页面
- 公开知识库列表

## V1 不做

- 完整响应式移动体验
- 复杂后台管理
- 评论管理
- 用户管理后台
- 数据统计后台

## Web 最小闭环

```text
登录 → 查看文档列表 → 新建文档 → Android 端同步看到
```

---

## 7. V1 后端范围

后端是 Android 和 Web 的统一数据中心。

## V1 必做 API 能力

### Auth

- 注册
- 登录
- JWT 鉴权
- 获取当前用户信息

### Document

- 创建文档
- 获取我的文档列表
- 获取文档详情
- 更新文档
- 删除文档
- 获取公开文档列表

### Todo

- 创建待办
- 获取待办列表
- 更新待办状态
- 删除待办

### AI

- 普通提问
- 总结当前文档
- 生成 Markdown 草稿

## V1 可选 API 能力

- 标签管理
- 文档搜索
- 文档公开/取消公开
- AI 回答保存为文档

## V1 不做 API 能力

- 付费订阅
- 完整社区评论
- 向量检索
- CLI Token 管理
- 管理后台权限

---

## 8. V1 数据模型范围

V1 至少需要以下核心表：

| 表 | 作用 |
|---|---|
| users | 用户账号 |
| documents | Markdown 文档 |
| todos | 待办任务 |
| ai_conversations | AI 对话，可选 |
| ai_messages | AI 消息，可选 |

V1 可以暂缓：

| 表 | 暂缓原因 |
|---|---|
| comments | V3 公开知识库互动再做 |
| likes | V3 再做 |
| follows | V3 再做 |
| vector_chunks | V2/V3 RAG 再做 |
| subscriptions | 商业化阶段再做 |

---

## 9. V1 明确不做清单

为了避免范围膨胀，V1 明确不做：

- Obsidian 完整复刻
- 插件系统
- 双链和知识图谱
- 多人协作
- 完整社区评论
- 复杂公开主页
- 向量数据库 RAG
- CLI 工具
- 付费系统
- 通知推送
- 复杂数据统计

这些功能不是不重要，而是应该在 V2/V3/V4 逐步引入。

---

## 10. V1 验收标准

V1 完成时，应该可以演示以下流程：

### 流程 1：Android 文档闭环

```text
注册/登录 → 进入 Markdown → 新建文档 → 输入 Markdown → 保存 → 查看渲染 → 编辑文档
```

### 流程 2：Web 与 Android 同步

```text
Web 登录 → 创建文档 → Android 刷新 → 看到同一篇文档
```

### 流程 3：待办闭环

```text
进入待办 → 创建任务 → 标记完成 → 查看完成状态
```

### 流程 4：AI 问答闭环

```text
进入 AI → 输入问题 → 获得回答
```

### 流程 5：文档总结闭环

```text
打开 Markdown → 点击 AI 总结 → 看到摘要 → 复制或保存
```

### 流程 6：知识库浏览闭环

```text
进入知识库 → 查看公开/示例文档 → 打开并渲染 Markdown
```

---

## 11. 当前结论

MKM V1 采用 **Android 优先增强 MVP**：

```text
Android 是主产品端
Backend 是数据和 AI 能力中心
Web 是辅助管理端
CLI 暂缓到 V4
```

V1 的核心是把 5 个一级入口跑通：

```text
Markdown    待办    AI    知识库    我的
```

其中 Markdown 是基础，AI 是差异化，待办是行动承接，知识库是未来社区入口，我的是账号和设置中心。
