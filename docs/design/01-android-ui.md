# MKM Android UI 设计规范

## 1. 文档目标

本文档定义 MKM Android 端 V1 所有页面的 UI 逻辑、状态流转和交互规范。

Android 端是 MKM 的主产品端，V1 的所有核心功能都需要在 Android 上完整可用。

---

## 2. 整体导航结构

## 2.1 全局入口

```text
LoginActivity（启动页）
    └── MainActivity（主页）
        ├── MarkdownFragment     Tab 1 文档
        ├── TodoFragment         Tab 2 待办
        ├── AiAssistantFragment  Tab 3 AI（中间）
        ├── KnowledgeFragment    Tab 4 知识库
        └── ProfileFragment      Tab 5 我的
```

## 2.2 页面跳转关系

```text
LoginActivity
    → MainActivity（登录成功）

MainActivity / MarkdownFragment
    → DocumentDetailActivity（点击文档或新建）

DocumentDetailActivity
    → （弹窗）AI 总结结果
    → （弹窗）提取待办草稿 → 写入 todos
    → （跳转）DocumentDetailActivity 编辑模式

AiAssistantFragment
    → AiProviderSettingsActivity（未配置时引导）
    → DocumentDetailActivity（保存生成的 Markdown）
    → TodoFragment（提取待办后跳转）

ProfileFragment
    → AiProviderSettingsActivity（AI 设置入口）
    → LoginActivity（退出登录）
```

## 3. 颜色与主题建议

V1 建议使用简洁深色/浅色主题，不要过于花哨。

主色：

```text
#3F51B5  Indigo（图标、按钮、强调）
#009688  Teal（次要强调、AI 相关元素）
#FFFFFF  白（背景）
#F5F5F5  浅灰（分组背景）
#212121  主文字
#757575  次要文字
```

## 4. LoginActivity

## 4.1 页面结构

```text
┌────────────────────────────┐
│                            │
│         MKM Logo           │
│                            │
│    用户名输入框             │
│    密码输入框               │
│                            │
│    [登录]  [注册]           │
│                            │
│    错误提示（可选）         │
│    Loading 指示器           │
│                            │
└────────────────────────────┘
```

## 4.2 状态说明

| 状态 | UI 表现 |
|---|---|
| 默认 | 输入框可交互，按钮可点击 |
| 加载中 | 按钮 disabled，ProgressBar 显示 |
| 登录失败 | 显示 Snackbar 错误提示 |
| 登录成功 | 跳转 MainActivity |

## 4.3 交互细节

- 两个输入框支持 Enter 键触发登录
- 密码输入框支持眼睛图标切换明文/密文
- 键盘弹起时页面上移，不遮挡按钮
- 登录和注册共用输入框，切换按钮文字不同

## 5. MainActivity

## 5.1 页面结构

```text
┌────────────────────────────┐
│  Fragment 内容区域          │
│                            │
│                            │
│                            │
├────────────────────────────┤
│ 文档 │ 待办 │ AI │ 知识 │ 我 │  ← BottomNavigationView
└────────────────────────────┘
```

## 5.2 底部导航图标建议

| Tab | 图标建议 | 标签 |
|---|---|---|
| Markdown | ic_description / 文件图标 | 文档 |
| Todo | ic_check_box / 任务图标 | 待办 |
| AI | ic_smart_toy / AI 图标 | AI |
| Knowledge | ic_library_books / 书架图标 | 知识库 |
| Profile | ic_person / 人物图标 | 我的 |

AI 图标可以用深色圆形背景做突出效果，类似微信中间的"+"按钮视觉感。

## 5.3 交互细节

- 切换 Tab 不重置 Fragment 状态
- 首次打开默认进入 MarkdownFragment
- 深链接：从 AI 生成文档跳转到 DocumentDetailActivity 后，返回回到 MarkdownFragment

## 6. MarkdownFragment（文档列表）

```text

│ 🔍 搜索文档...              │  ← SearchView（V1 可选）

├────────────────────────────┤

│   ...                      │
│                            │
│                      [+]   │  ← FAB 新建
└────────────────────────────┘
```

## 6.2 文档卡片

```text

│ Spring Boot 学习笔记         │  ← title
```


## 6.3 状态说明

| 状态 | UI 表现 |
|---|---|
| 加载中 | 顶部 LinearProgressIndicator |

| 空状态 | 插图 + "还没有文档，点击 + 新建" |

| 网络错误无缓存 | 空状态 + 重试按钮 |

```text
├────────────────────────────┤
```

## 7.2 编辑模式页面结构

```text

├────────────────────────────┤
```

进入即为编辑模式：

```text

```

## 7.4 状态说明

| 状态 | UI 表现 |
|---|---|
| 加载文档 | 顶部 ProgressBar |
| 预览模式 | Markwon 渲染 + 底部操作栏 |
| 编辑模式 | 输入框，底部操作栏隐藏 |
| 保存中 | 保存按钮 disabled + ProgressBar |
| 保存成功 | Snackbar + 回到预览模式 |
| 保存失败 | Snackbar 错误提示 |

## 13. 全局状态和通用 UI 规范

## 13.1 Loading 规范

| 场景 | 组件 |
|---|---|
| 页面初始加载 | 顶部 LinearProgressIndicator |
| 局部数据加载 | 单个 CircularProgressIndicator |
| 按钮操作中 | 按钮 disabled + 文字变"保存中..." |
| AI 生成中 | 消息区域 typing 动画气泡 |

## 13.2 错误提示规范

| 场景 | 组件 |
|---|---|
| 操作失败 | Snackbar |
| 网络异常 | Snackbar + 重试 Action |
| Token 过期 | Dialog 提示 → 跳转登录 |
| 字段校验失败 | 输入框下方 TextInputLayout 错误提示 |

## 13.3 空状态规范

所有列表页面需要有空状态设计：

```text
插图（可用系统图标代替）
标题
副文字
可选操作按钮
```

## 13.4 Snackbar 规范

```text
成功：短文字 + 无 Action（2s 自动消失）
警告：短文字 + 可选 Action（4s）
错误：错误原因 + "重试" Action（长显示）
```

## 14. 当前结论

MKM Android V1 页面结构确定：

```text
LoginActivity
MainActivity
├── MarkdownFragment       文档列表 + 搜索 + FAB
├── TodoFragment           待办列表 + Tab切换 + FAB
├── AiAssistantFragment    AI 聊天 + 快捷操作 + Markdown 渲染
├── KnowledgeFragment      公开/示例知识库列表
└── ProfileFragment        用户信息 + AI设置 + 退出
DocumentDetailActivity     预览 + 编辑 + AI 总结弹窗 + 提取待办弹窗
AiProviderSettingsActivity AI Key 管理 + 添加模型
```

核心 UI 原则：

```text
BottomSheet 处理弹窗式结果展示（AI 总结、待办选择）
LinearProgressIndicator 处理加载
Snackbar 处理操作反馈
空状态要友好
AI 页面消息气泡支持 Markdown 渲染
```
