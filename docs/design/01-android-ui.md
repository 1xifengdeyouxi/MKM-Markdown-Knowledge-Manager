# MKM Android UI 设计规范

## 1. 文档目标

本文档定义 MKM Android 端 V1 所有页面的 UI 逻辑、状态流转和交互规范。

Android 端是 MKM 的主产品端，V1 的所有核心功能都需要在 Android 上完整可用。

---

## 2. 整体导航结构

## 2.1 全局入口

```text
LoginActivity（启动页）
    └── MainActivity（主页）
        ├── MarkdownFragment     Tab 1 文档
        ├── TodoFragment         Tab 2 待办
        ├── AiAssistantFragment  Tab 3 AI（中间）
        ├── KnowledgeFragment    Tab 4 知识库
        └── ProfileFragment      Tab 5 我的
```

## 2.2 页面跳转关系

```text
LoginActivity
    → MainActivity（登录成功）

MainActivity / MarkdownFragment
    → DocumentDetailActivity（点击文档或新建）

DocumentDetailActivity
    → （弹窗）AI 总结结果
    → （弹窗）提取待办草稿 → 写入 todos
    → （跳转）DocumentDetailActivity 编辑模式

AiAssistantFragment
    → AiProviderSettingsActivity（未配置时引导）
    → DocumentDetailActivity（保存生成的 Markdown）
    → TodoFragment（提取待办后跳转）

ProfileFragment
    → AiProviderSettingsActivity（AI 设置入口）
    → LoginActivity（退出登录）
```

主色：

```text
#3F51B5  Indigo（图标、按钮、强调）
#009688  Teal（次要强调、AI 相关元素）
#FFFFFF  白（背景）

#212121  主文字
#757575  次要文字

## 4. LoginActivity

## 4.1 页面结构

```text
┌────────────────────────────┐
│                            │
│         MKM Logo           │
│                            │
│    用户名输入框             │
│    密码输入框               │
│                            │
│    [登录]  [注册]           │
│                            │
│    错误提示（可选）         │
│    Loading 指示器           │
│                            │
└────────────────────────────┘
```

## 4.2 状态说明

| 状态 | UI 表现 |
|---|---|
| 默认 | 输入框可交互，按钮可点击 |
| 加载中 | 按钮 disabled，ProgressBar 显示 |
| 登录失败 | 显示 Snackbar 错误提示 |
| 登录成功 | 跳转 MainActivity |

## 4.3 交互细节

- 两个输入框支持 Enter 键触发登录
- 密码输入框支持眼睛图标切换明文/密文
- 键盘弹起时页面上移，不遮挡按钮
- 登录和注册共用输入框，切换按钮文字不同

## 6. MarkdownFragment（文档首页）

## 6.1 页面定位

MarkdownFragment 是 MKM 的默认首页，定位不是单纯的文档列表，而是当前仓库的文档工作台。

核心信息：

```text
当前仓库 / 项目名
最近访问文档
目录结构
文档搜索
列表排序与筛选
```

## 6.2 页面结构

```text
┌────────────────────────────┐
│ 👤  我的知识库      🔍  ⋮   │  ← Toolbar：头像 + 项目名 + 搜索 + 更多
├────────────────────────────┤
│ [最近]  [目录]        [≡]  │  ← Tab + 列表设置
├────────────────────────────┤
│ 文档卡片                   │
│   标题                     │
│   更新时间  标签            │
├────────────────────────────┤
│ 文档卡片                   │
│   ...                      │
│                            │
│                      [+]   │  ← FAB 新建
└────────────────────────────┘
```

## 6.3 Toolbar 设计

Toolbar 左侧显示：

```text
头像 / 仓库图标 + 当前仓库名称
```

Toolbar 右侧显示：

```text
搜索图标 + 更多菜单
```

| 区域 | 说明 |
|---|---|
| 头像 | 可以显示用户头像；未登录时显示本地仓库图标 |
| 项目名 | 当前仓库名称，例如“我的知识库” |
| 搜索 | 点击进入搜索状态 |
| 更多 | 仓库信息、同步状态、导入 Markdown、设置 |

更多菜单建议：

```text
仓库信息
导入 Markdown
同步状态
设置
```

## 6.4 最近 / 目录 Tab

页面下方使用两个主 Tab：

```text
最近    目录
```

| Tab | 作用 |
|---|---|
| 最近 | 展示最近创建、最近编辑、最近打开的 Markdown 文档 |
| 目录 | 展示当前仓库的目录结构 |

V1 可以先实现“最近”列表，“目录”页面展示仓库目录树的基础版本。

## 6.7 搜索状态

点击搜索图标后，Toolbar 进入搜索模式：

```text
┌────────────────────────────┐
│ ←  搜索文档...        清除 │
└────────────────────────────┘
```

搜索范围：

```text
标题
标签
Markdown 正文（V1 可选）
```

搜索结果为空时：

```text
未找到相关文档
换个关键词试试
```

## 6.8 状态说明

| 状态 | UI 表现 |
|---|---|
| 加载中 | 顶部 LinearProgressIndicator |
| 有数据 | 展示 RecyclerView，支持列表/宫格切换 |
| 空状态 | 插图 + “还没有文档，点击 + 新建” |
| 搜索无结果 | 空状态 + “未找到相关文档” |
| 网络错误有缓存 | 展示缓存 + Snackbar 提示“同步失败” |
| 网络错误无缓存 | 空状态 + 重试按钮 |

## 6.9 下拉刷新

支持 SwipeRefreshLayout 下拉刷新。

如果当前是本地模式，下拉刷新用于重新扫描仓库目录；如果已开启云同步，下拉刷新用于触发同步。

## 7.5 Markdown 辅助工具栏

工具栏固定在软键盘上方，随键盘弹出和收起。

```text
┌────────────────────────────────────────┐
│ B  I  S  H1  H2  ```  > [] ()  🖼  ··· │
└────────────────────────────────────────┘
```

| 按钮 | 作用 |
|---|---|
| B | 加粗：`**文字**` |
| I | 斜体：`*文字*` |
| S | 删除线：`~~文字~~` |
| H1 | 一级标题：`# ` |
| H2 | 二级标题：`## ` |
| ` ``` ` | 代码块 |
| > | 引用块：`> ` |
| [] | 任务项：`- [ ] ` |
| () | 链接：`[](url)` |
| 🖼 | 插入图片（V2） |
| ··· | 展开更多（表格、水平线等） |

点击 `···` 展开更多：

```text
┌──────────────────────────────┐
│ 表格  水平线  有序列表        │
│ 无序列表  代码行  公式（V2）   │
└──────────────────────────────┘
```

## 7.7 状态说明

| 状态 | UI 表现 |
|---|---|
| 加载文档 | 顶部 ProgressBar |
| 预览模式 | Markwon 渲染 + 底部操作栏 |
| 编辑模式 | 标题输入框 + 标签行 + 内容输入框 + 辅助工具栏 |

| 保存中 | 保存按钮 disabled + ProgressBar |
| 保存成功 | Snackbar + 回到预览模式 |
| 保存失败 | Snackbar 错误提示 |

## 13. 全局状态和通用 UI 规范

## 13.1 Loading 规范

| 场景 | 组件 |
|---|---|
| 页面初始加载 | 顶部 LinearProgressIndicator |
| 局部数据加载 | 单个 CircularProgressIndicator |
| 按钮操作中 | 按钮 disabled + 文字变"保存中..." |
| AI 生成中 | 消息区域 typing 动画气泡 |

## 13.2 错误提示规范

| 场景 | 组件 |
|---|---|
| 操作失败 | Snackbar |
| 网络异常 | Snackbar + 重试 Action |
| Token 过期 | Dialog 提示 → 跳转登录 |
| 字段校验失败 | 输入框下方 TextInputLayout 错误提示 |

## 13.3 空状态规范

所有列表页面需要有空状态设计：

```text
插图（可用系统图标代替）
标题
副文字
可选操作按钮
```

## 13.4 Snackbar 规范

```text
成功：短文字 + 无 Action（2s 自动消失）
警告：短文字 + 可选 Action（4s）
错误：错误原因 + "重试" Action（长显示）
```

## 14. 当前结论

MKM Android V1 页面结构确定：

```text
LoginActivity
MainActivity
├── MarkdownFragment       文档列表 + 搜索 + FAB
├── TodoFragment           待办列表 + Tab切换 + FAB
├── AiAssistantFragment    AI 聊天 + 快捷操作 + Markdown 渲染
├── KnowledgeFragment      公开/示例知识库列表
└── ProfileFragment        用户信息 + AI设置 + 退出
DocumentDetailActivity     预览 + 编辑 + AI 总结弹窗 + 提取待办弹窗
AiProviderSettingsActivity AI Key 管理 + 添加模型
```

核心 UI 原则：

```text
BottomSheet 处理弹窗式结果展示（AI 总结、待办选择）
LinearProgressIndicator 处理加载
Snackbar 处理操作反馈
空状态要友好
AI 页面消息气泡支持 Markdown 渲染
```
